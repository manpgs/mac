NAME

kuncd — The Kernel User Notification Center daemon.

DESCRIPTION

The Kernel User Notification Center daemon handles requests by software executing in the kernel to display notices and alerts to the user. The daemon also handles kernel reqeusts to execute user-space helper tools.

The daemon is launched by launchd(8) when a request has been received from the kernel and will exit 15 seconds after the last request has been processed.

The daemon cannot be used directly by the user.

SIGNALS

SIGTERM
This signal will result in the daemon unregistering itself from launchd and exiting. Once it is unregistered, it will not run again until it is reregistered.

SEE ALSO

launchd(8)

HISTORY

The kuncd daemon was introduced in Mac OS X 10.3.

NOTES

This daemon and its current behavior may change without notice. Do not rely on its existence or its behavior. Consider it an unsupported command.
